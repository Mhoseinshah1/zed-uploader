# درگاه‌های پرداخت و استارز تلگرام

پنل → «درگاه‌ها». هر درگاه مستقل پیکربندی می‌شود؛ درگاه فقط وقتی به کاربر
نمایش داده می‌شود که «فعال + پیکربندی کامل» باشد. اگر هیچ درگاهی آماده نباشد
دکمهٔ پرداخت آنلاین مخفی می‌شود (کارت‌به‌کارت همیشه هست).

## CentralPay
کلیدهای `getLink` و `verify` را از پنل وارد کنید، یا در `.env`
(`CENTRALPAY_GETLINK_KEY` / `CENTRALPAY_VERIFY_KEY`) — مقدار پنل مقدم است و
env صرفاً fallback سازگاری است.

## زرین‌پال (Zarinpal)
مرچنت (Merchant ID) را وارد کنید؛ «سندباکس» برای تست. بازگشت:
`GET /pay/zarinpal/return`. مبلغ‌ها تومان.

## زیبال (Zibal)
مرچنت را وارد کنید؛ «سندباکس» = مرچنت تستی `zibal`. توجه: API زیبال ریالی است
و تبدیل تومان↔ریال (×۱۰/÷۱۰) خودکار انجام می‌شود.

## استارز تلگرام (XTR)
پنل → «پلن‌ها» → «قیمت استارز» را پر کنید تا دکمهٔ ⭐ در خرید پلن ظاهر شود.
پرداخت idempotent است (شناسهٔ charge تلگرام کلید یکتاست؛ دوبار اعمال نمی‌شود).

## کارت‌به‌کارت
پنل → «تنظیمات» → شماره کارت/نام دارنده + حداقل شارژ. رسیدها به مالک‌ها ارسال
و از پنل/ربات تأیید می‌شوند (تأیید idempotent؛ شارژ فقط یک‌بار).

**ایمنی پول:** verify هر درگاه idempotent است (قفل ردیف + وضعیت)، مبلغ
نامنطبق هرگز شارژ نمی‌شود، و همهٔ تغییرات موجودی فقط از WalletService (دفتر
کل) عبور می‌کند: `SUM(tx) == balance`.
