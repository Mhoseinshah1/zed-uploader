# چک‌لیست راه‌اندازی نهایی ZedUploader (کارهای روی سرور)

> این‌ها کارهایی هستند که Claude Code نمی‌تواند انجام دهد و فقط خودت روی سرور و با حساب‌های واقعی باید انجام دهی. به‌ترتیب برو.
>
> ✅ **تأیید شده با کد** (ممیزی نهایی QA): همهٔ مسیرها/دستورهای این چک‌لیست با
> مخزن مطابقت دارند. head فعلی migration = `0026_panel_roles`. گزارش کامل
> ممیزی: `docs/LAUNCH_READINESS.md`.

## گام ۱ — استقرار آخرین نسخه
- [ ] یک بکاپ کامل بگیر (قبل از هر کاری):
  ```
  cd ~/zed-uploader
  docker compose exec -T db pg_dump -U <POSTGRES_USER> <POSTGRES_DB> > ~/backup-before-golive.sql
  ```
- [ ] آخرین کد را بکش و migrationها را اعمال کن:
  ```
  git pull && sudo bash update.sh
  ```
- [ ] مطمئن شو migration به head رسیده و سرویس‌ها بالا هستند:
  ```
  docker compose run --rm api alembic current   # باید بنویسد: 0026_panel_roles (head)
  curl -sI https://zedproxy.online/health       # باید 200 بدهد ({"status":"ok"})
  ```
  نکته: `update.sh` خودش قبل از pull یک بکاپ خودکار هم می‌گیرد (بکاپ دستی بالا،
  احتیاط دوبل است — نگهش دار).

## گام ۲ — پیکربندی از پنل مادر
وارد `https://zedproxy.online/panel` شو و این‌ها را ست کن:
- [ ] **درگاه‌ها** (بخش Payment Providers): کلید واقعی هر درگاه که می‌خواهی (CentralPay / Zarinpal / Zibal) — حالت sandbox را **خاموش** کن، فقط درگاه‌هایی که کلید واقعی دادی را enable کن.
- [ ] **Telegram Stars**: اگر می‌خواهی، toggle سراسری را روشن کن.
- [ ] **پلن‌ها**: قیمت هر پلن + قیمت Stars هر پلن (اگر Stars روشن است).
- [ ] **تنظیمات پرداخت**: شماره کارت واقعی، `topup_min`، و toggle کارت‌به‌کارت.
- [ ] **قیمت ساخت/اجارهٔ ربات** (برای نماینده‌ها) در پنل مادر.
- [ ] **کانال‌های عضویت اجباری**، **متن‌های بات**، و **منوی دستورات**.

## گام ۳ — تست پرداخت واقعی (حیاتی — از رویش نپر)
برای **هر درگاهی که فعال کردی**، با **مبلغ خیلی کم** (مثلاً هزار تومان) یک خرید واقعی انجام بده و چشمی چک کن:
- [ ] CentralPay: پول یک‌بار در کیف پول نشست؟ فاکتور صادر شد؟
- [ ] Zarinpal: همان.
- [ ] Zibal: همان.
- [ ] Telegram Stars: خرید پلن یک‌بار فعال شد؟
- [ ] کارت‌به‌کارت: رسید ثبت و بعد از تأیید ادمین، شارژ شد؟
- [ ] برای **یک بات نماینده** هم یک خرید تست کن تا مطمئن شوی به بات درست می‌رسد (نه بات تو).
- [ ] در همهٔ موارد: در پنل → کیف پول/ledger/فاکتور همه درست و **بدون دوبار شارژ** باشند.

## گام ۴ — تست ساخت ربات (کارخانه)
- [ ] از ربات مادر، با یک توکن تست BotFather، مسیر «ساخت ربات» را کامل برو: پرداخت → ساخت → بات زنده شد؟ webhook ست شد؟ لاگین پنل مشتری کار کرد؟
- [ ] با یک لاگین نماینده وارد پنل شو و مطمئن شو **فقط دادهٔ خودش** را می‌بیند و به قابلیت‌های مادر (ساخت ربات، قیمت‌گذاری، آمار کل) دسترسی ندارد.

## گام ۵ — سخت‌سازی سرور
- [ ] **بکاپ خودکار** را از پنل فعال کن و یک بکاپ دستی بگیر تا مطمئن شوی کار می‌کند.
  (صفحهٔ «پشتیبان‌گیری» فقط با لاگین **سوپرادمین** پلتفرم دیده می‌شود.)
- [ ] **تست restore**: یک بکاپ را روی یک محیط تست (نه production!) restore کن تا مطمئن شوی بکاپ‌هایت واقعاً قابل بازگردانی‌اند.
- [ ] تمدید خودکار SSL را تست کن:
  ```
  sudo certbot renew --dry-run
  ```
- [ ] فایروال Hetzner: فقط پورت‌های **۸۰ / ۴۴۳ / ۲۲** باز باشند؛ بقیه (مثل ۸۰۸۰ تست) بسته.
- [ ] یک مانیتورینگ ساده روی `/health` بگذار (مثلاً یک cron که هر چند دقیقه چک کند و اگر قطع شد به تو خبر دهد).

## گام ۶ — تجاری‌سازی (اگر سورس/نمایندگی می‌فروشی)
- [ ] سرور activation را جای مستقل deploy کن و چند license key صادر کن.
- [ ] یک کانال/آیدی **پشتیبانی** برای نماینده‌ها بساز.
- [ ] **شرایط استفاده** و **سیاست بازگشت وجه** بنویس (چون پول می‌گیری).
- [ ] سیستم گزارش تخلف را روشن نگه دار و به گزارش‌ها رسیدگی کن (چون کاربران آپلود می‌کنند).

---

## نکتهٔ مهم
مهم‌ترین و حساس‌ترین گام، **گام ۳ (تست پرداخت واقعی)** است. همهٔ تست‌های کد ما mock بوده‌اند؛ تنها راه مطمئن‌شدن اینکه پول در دنیای واقعی درست جابه‌جا می‌شود، همین تست کوچک واقعی است. تا این را نکرده‌ای، به هیچ مشتری واقعی نفروش.
