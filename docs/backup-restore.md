# پشتیبان‌گیری و بازیابی

پنل → «پشتیبان»: پشتیبان دستی، زمان‌بندی روزانه/هفتگی، نگه‌داری N نسخهٔ آخر،
دانلود/حذف. `update.sh` هم قبل از هر به‌روزرسانی پشتیبان می‌گیرد.

> ⚠️ **بازیابی مخرب و غیرقابل بازگشت است**: دیتابیس فعلی کاملاً با فایل
> پشتیبان جایگزین می‌شود. فقط مالک انجام دهد؛ قبل از آن یک پشتیبان تازه
> بگیرید؛ برای تأیید باید نام دقیق فایل تایپ شود؛ بهتر است bot و worker را
> موقتاً خاموش کنید (`docker compose stop bot worker`).

بازیابی دستی از خط فرمان:

```bash
cat /backups/FILE.sql | docker compose exec -T db psql -U "$POSTGRES_USER" "$POSTGRES_DB"
```
