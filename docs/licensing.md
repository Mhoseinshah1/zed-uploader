# لایسنس و فعال‌سازی

## مدل
- `LICENSE_DISABLED=true` (پیش‌فرض) یا نبود key/server ⇒ **بدون لایسنس، همهٔ
  قابلیت‌ها آزاد** (حالت توسعه/استقرارهای فعلی).
- با لایسنس نامعتبر/منقضی/باطل: فقط اقدامات فروش جدید (پرداخت آنلاین، خرید
  پلن، استارز، ارسال همگانی) با پیام فارسی غیرفعال می‌شوند. **تحویل فایل،
  مشاهدهٔ پنل و همهٔ داده‌ها دست‌نخورده می‌مانند** و با معتبر شدن دوباره،
  همه‌چیز برمی‌گردد.
- آفلاین‌بودن سرور فعال‌سازی: تا `LICENSE_GRACE_DAYS` (پیش‌فرض ۷ روز) از آخرین
  تأیید موفق، همه‌چیز فعال می‌ماند (دورهٔ مدارا).
- اثر انگشت سرور: `sha256(machine-id + ":" + مسیر نصب)` — پایدار بین ری‌استارت‌ها.

## فعال‌سازی مشتری
`.env`: `LICENSE_DISABLED=false` + `LICENSE_KEY` + `LICENSE_SERVER_URL` →
پنل → «لایسنس» → فعال‌سازی. ورکر روزانه وضعیت را تازه می‌کند.

## صدور کلید (فروشنده)
سرویس اختیاری `activation_server/` را اجرا کنید و:

```bash
python -m activation_server.issue --key CUST-1 --seats 2 --days 365
python -m activation_server.issue --list
python -m activation_server.issue --key CUST-1 --revoke
```

هر fingerprint یک صندلی مصرف می‌کند؛ فعال‌سازی مجدد همان سرور صندلی جدید
نمی‌گیرد؛ بیش از سقف رد می‌شود. جزئیات: `activation_server/README.md`.
